作为一个创意总监，我不只是想帮你把代码写通，我还要让这个功能在视觉和交互上显得“高级”。

目前大多数 Chrome 扩展的工具栏都显得很拥挤。既然 DashNote 是一个笔记类工具，**“呼吸感”和“确定感”**至关重要。

以下是针对 `Select All` 功能的直接执行方案：

### 1. HTML 结构 (`sidepanel.html`)

在 `id="selectedCount"` 容器旁边，我们加入这个切换按钮。建议给它一个通用的类名以便统一样式。

```html
<!-- 寻找你的 toolbar 区域 -->
<div id="selectionToolbar" class="selection-toolbar">
  <div class="toolbar-left">
    <span id="selectedCount">0</span>
    <!-- 新增全选按钮 -->
    <button id="btnSelectAll" class="btn-ghost">全选</button>
  </div>
  <div class="toolbar-actions">
    <button id="btnCopySelected">复制</button>
    <button id="btnDeleteSelected">删除</button>
  </div>
</div>
```

### 2. 视觉设计与样式建议 (CSS)

不要使用原生的灰色按钮。为了体现“专业感”，我建议使用**克制的 Indigo（靛蓝色）**作为激活态。

```css
/* 建议添加到你的 CSS 文件中 */
#btnSelectAll {
  margin-left: 12px;
  padding: 4px 8px;
  font-size: 12px;
  font-weight: 600;
  color: #6366f1; /* Indigo 500 */
  background: rgba(99, 102, 241, 0.08);
  border: none;
  border-radius: 4px;
  cursor: pointer;
  transition: all 0.2s ease;
}

#btnSelectAll:hover {
  background: rgba(99, 102, 241, 0.15);
  color: #4f46e5;
}

/* 当处于“取消全选”状态时的微调 */
#btnSelectAll.is-active {
  color: #ef4444; /* Rose 500 - 提示这是一个反向操作 */
  background: rgba(239, 68, 68, 0.08);
}
```

### 3. 逻辑实现 (`sidepanel.js`)

这是功能的核心。我们要确保它能感知当前可见的所有笔记。

```javascript
// 1. 初始化 DOM 引用
const btnSelectAll = document.getElementById('btnSelectAll');

// 2. 全选函数
function selectAllNotes() {
    // 获取当前页面上所有可见的笔记卡片 ID (假设你的卡片有 data-id 属性)
    const allNoteCards = document.querySelectorAll('.note-card'); 
    allNoteCards.forEach(card => {
        const noteId = card.dataset.id;
        selectedNotes.add(noteId);
        card.classList.add('selected'); // 视觉反馈
    });
    updateSelectionUI();
}

// 3. 取消全选函数
function deselectAllNotes() {
    selectedNotes.clear();
    const allNoteCards = document.querySelectorAll('.note-card');
    allNoteCards.forEach(card => {
        card.classList.remove('selected');
    });
    updateSelectionUI();
}

// 4. 切换逻辑
function toggleSelectAll() {
    const allNoteCards = document.querySelectorAll('.note-card');
    const isAllSelected = selectedNotes.size >= allNoteCards.length && allNoteCards.length > 0;

    if (isAllSelected) {
        deselectAllNotes();
        btnSelectAll.textContent = '全选';
        btnSelectAll.classList.remove('is-active');
    } else {
        selectAllNotes();
        btnSelectAll.textContent = '取消全选';
        btnSelectAll.classList.add('is-active');
    }
}

// 5. 事件监听与重置逻辑
btnSelectAll.addEventListener('click', toggleSelectAll);

// 在你现有的 toggleSelectMode 函数中，当退出选择模式时：
function exitSelectMode() {
    // ... 现有逻辑
    deselectAllNotes();
    btnSelectAll.textContent = '全选';
    btnSelectAll.classList.remove('is-active');
}

// 记得在你的 updateSelectionUI 中同步更新按钮文字
// 如果用户手动一个个勾选到了全部，按钮也应该变成“取消全选”
function updateSelectionUI() {
    const count = selectedNotes.size;
    document.getElementById('selectedCount').textContent = `${count} 项已选择`;
    
    const allNoteCards = document.querySelectorAll('.note-card');
    if (count > 0 && count === allNoteCards.length) {
        btnSelectAll.textContent = '取消全选';
        btnSelectAll.classList.add('is-active');
    } else {
        btnSelectAll.textContent = '全选';
        btnSelectAll.classList.remove('is-active');
    }
}
```

### 创意总监的额外建议 (Bold Advice)

1.  **动效加分**：当点击“全选”时，不要让所有卡片瞬间变色。在 CSS 中给 `.note-card.selected` 加上一个微小的弹性缩放效果：`transform: scale(0.98); transition: transform 0.2s cubic-bezier(0.34, 1.56, 0.64, 1);`。这会让批量操作感觉非常有手感。
2.  **空状态处理**：如果当前搜索结果为空（没有卡片），记得禁用 `btnSelectAll`。
    `btnSelectAll.disabled = allNoteCards.length === 0;`
3.  **对比度**：确保 `selectedCount` 的颜色比按钮稍淡（推荐 `#6B7280`），让用户的视觉焦点落在操作按钮（全选、复制、删除）上，而不是统计数字上。

这个实现方式不仅满足了你的功能需求，还赋予了插件一种“原生 App”的高级感。
